  • Imports
    • Alphabetized
  • Classes
    • Names should be no more than 3 to 4 words.
    • Distinct words in the name should be capitalized
      • Example: WorldMap
    • Instances should be named with camel case
      • Example: worldMap
    • Every class should be in its own source file with a .java extension.
    • Encapsulation should always be on your mind!
  • Spacing
    • All operators should be separated by a single space.
      • Example: float z = x * x + y * y;
    • All curly brackets should be on there own line
      • Example:
        • class SomeClass
        {
        // ...
         }
      • Example:
        • for (int i = 0; i < 2000; i++)
        {
        // ...
         }
    • Statements that use curly brackets (for, while, etc) should be written with spaces as follows:
      • For loops:
        • for (int i = 0; i < anotherInt; i++)
        {
        // ...
         }
      • While loops:
        • while (i < someInt)
        {
        // ...
         }
      • Do-while loops:
        • do
        {
        // ...
         }
        while (42 != meaningOfLife);
      • Try-catch blocks:
        • try
        {
        // try to do something...
         }
        catch (Exception e)
        {
        // do something for this exceptional case...
         }
    • Lines should be limited to 79 characters when possible. However, exceeding this limit is fine if the content is more readable on a single line.
  • Comments

      * @version 1.6 (current version number of this program)

      }

      * Short one line description.
      *
      * Longer description (if needed)
      *
      * @param Description of this parameter.
      * @return Description of what this method returns.
